## Активный залог

Пассивный залог скрывает действующее лицо и усложняет понимание.

| ❌ Плохо | ✅ Хорошо |
|----------|-----------|
| Запрос обрабатывается сервером | Сервер обрабатывает запрос |
| Данные были загружены | Система загрузила данные |
| Решение было принято | Команда приняла решение |
| Ошибка была обнаружена | Тест обнаружил ошибку |

**Исключение:** пассив допустим, когда деятель неизвестен или неважен («Письмо отправлено»).

---

## Утвердительная форма

Отрицания требуют дополнительного мыслительного усилия.

| ❌ Плохо | ✅ Хорошо |
|----------|-----------|
| Не забудьте сохранить | Сохраните |
| Он не помнил | Он забыл |
| Не является невозможным | Возможно |
| Не следует игнорировать | Учитывайте |

**Двойное отрицание — враг ясности.** «Нельзя не отметить» → «Стоит отметить» или убрать.

---

## Конкретика вместо абстракций

Абстрактные слова ничего не сообщают читателю.

| ❌ Плохо | ✅ Хорошо |
|----------|-----------|
| Работает быстро | Отвечает за 50 мс |
| Много пользователей | 12 000 пользователей |
| Значительно улучшили | Ускорили в 3 раза |
| Недавно обновили | Обновили 15 января |
| Хорошая производительность | 10 000 RPS при p99 < 100 мс |

**Правило:** если можно заменить слово числом, датой или фактом — замени.

---

## Глаголы вместо отглагольных существительных

Отглагольные существительные (-ание, -ение, -ция) утяжеляют текст.

| ❌ Плохо | ✅ Хорошо |
|----------|-----------|
| Произвести установку | Установить |
| Осуществить проверку | Проверить |
| Выполнить оптимизацию | Оптимизировать |
| Провести тестирование | Протестировать |
| Обеспечить функционирование | Запустить / Поддерживать |

**Тест:** если можно убрать «произвести/осуществить/выполнить» и поставить глагол — убирай.

---

## Удаление лишних слов

Каждое слово должно нести смысл. Если без слова предложение не теряет смысла — удаляй.

**Слова-паразиты:**
- «в принципе», «в целом», «как бы», «собственно говоря»
- «определённый», «соответствующий», «данный» (вместо «этот»)
- «достаточно», «довольно», «весьма»

**Вводные-пустышки:**

| ❌ Убрать | ✅ Оставить суть |
|-----------|------------------|
| Следует отметить, что система... | Система... |
| Необходимо подчеркнуть, что... | (просто написать факт) |
| Ни для кого не секрет, что... | (удалить полностью) |
| Как уже было сказано выше... | (удалить или дать ссылку) |

**Усилители без фактов:**

| ❌ Плохо | ✅ Хорошо |
|----------|-----------|
| Очень быстрый | Быстрый (или указать метрику) |
| Крайне важный | Важный (или объяснить почему) |
| Чрезвычайно полезный | Полезный (или показать пользу) |

## Канцеляризмы

Канцелярский язык отталкивает читателя.

| ❌ Канцелярит | ✅ Живой язык |
|---------------|---------------|
| В рамках данного проекта | В этом проекте |
| На сегодняшний день | Сейчас / Сегодня |
| В целях обеспечения | Чтобы обеспечить |
| Ввиду отсутствия | Без / Так как нет |
| Имеет место быть | Есть / Происходит |
| Вышеуказанный | Этот |
| Нижеследующий | Следующий |
| По причине того, что | Потому что |
| В связи с тем, что | Так как |
| При условии, что | Если |

## Структура абзаца

**Правило:** один абзац — одна мысль.

**Структура:**
1. Главное предложение — о чём абзац
2. Развитие мысли — детали, примеры, аргументы
3. Связь со следующим абзацем (опционально)

**Плохо:** абзац на 15 предложений обо всём сразу.

**Хорошо:** три абзаца по 3–5 предложений, каждый об одном аспекте.

---

## Порядок слов

Русский язык гибок, но порядок влияет на восприятие.

**Правило:** важное — в конец предложения. Конец запоминается лучше всего.

| Акцент на времени | Акцент на действии |
|-------------------|-------------------|
| Мы запустили сервис вчера | Вчера мы запустили сервис |

**Держи связанные слова рядом:**

| ❌ Плохо | ✅ Хорошо |
|----------|-----------|
| Он только планирует запустить тесты | Он планирует запустить только тесты |
| Система для пользователей с правами | Система с правами для пользователей |

---

## Однородность

Параллельные идеи выражай параллельными конструкциями.

**❌ Плохо:**
> Наша система обеспечивает: высокую скорость, надёжное хранение, а также помогает масштабироваться.

**✅ Хорошо:**
> Наша система обеспечивает скорость, надёжность и масштабируемость.

**Или:**
> Наша система работает быстро, хранит данные надёжно и легко масштабируется.

## Анти-паттерны AI-текста

Типичные признаки машинного текста. Избегай их.

**Пафос и громкие заявления:**

| ❌ AI-штамп | ✅ По-человечески |
|-------------|-------------------|
| Играет ключевую роль | Важен для... (объяснить почему) |
| Имеет колоссальное значение | Влияет на... (показать как) |
| Является неотъемлемой частью | Часть системы / Нужен для... |

**Маркетинговые прилагательные без фактов:**

| ❌ Удалить | ✅ Заменить фактом |
|------------|-------------------|
| Уникальный | Единственный в России / Первый с функцией X |
| Инновационный | Использует технологию Y |
| Революционный | Ускоряет процесс в 10 раз |
| Передовой | Соответствует стандарту Z |

**Деепричастия-затычки:**

| ❌ Плохо | ✅ Хорошо |
|----------|-----------|
| Обеспечивая надёжность | (убрать или объяснить как именно) |
| Демонстрируя возможности | Показывает возможность X: ... |
| Повышая эффективность | Экономит 2 часа в день |

**Шаблонные структуры:**
- Не начинай каждый абзац с «Также следует отметить...»
- Не заканчивай «Таким образом, можно сделать вывод...»
- Не используй списки, где достаточно предложения

## Пунктуация

**Запятая перед «и»:** ставь, если «и» соединяет два самостоятельных предложения.
> Сервер получил запрос, и обработка началась.

**Вводные конструкции:** выделяй запятыми.
> Как правило, система отвечает за 100 мс.

**Не ставь запятую** между подлежащим и сказуемым.

**Точка с запятой:** разделяет части сложного предложения, когда запятая уже использована внутри частей.

## Чек-лист редактирования

Перед отправкой текста проверь:

- [ ] Есть ли пассивный залог? → Замени на активный
- [ ] Есть ли отрицания? → Переформулируй в утверждение
- [ ] Есть ли абстракции? → Добавь конкретику
- [ ] Есть ли отглагольные существительные? → Замени глаголами
- [ ] Есть ли слова без смысловой нагрузки? → Удали
- [ ] Есть ли канцеляризмы? → Перепиши по-человечески
- [ ] Каждый абзац — об одном?
- [ ] Важное — в конце предложения?
- [ ] Нет AI-штампов?
